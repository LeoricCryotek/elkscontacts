Elks Contacts

Smarter member records for Odoo 19

Elks Contacts turns Odoo's Contacts app into a full Elks Lodge member system. It stores every CLMS field on res.partner, keeps members tagged with role toggles (Elk / Volunteer / Customer / Initiate / Guest), tracks officer terms, committee assignments, member history, suspensions, drops, and Return-to-Sender flags — and stays out of the way for non-member contacts.

One contact, many roles

Tag any contact as an Elk, Volunteer, Customer, Initiate, or Guest. The toggles reveal the matching views, smart buttons, and tabs — so a kitchen volunteer who happens to be a Life member sees both worlds on one record.

Smart-button bar — one-click member actions

The smart buttons turn the contact form into the daily operating console. Pay Dues sits at the front for reception; Mark RTS flags returned mail; Drop Member opens the drop wizard with the audit trail wired up. Every button respects the member toggles — non-Elks never see them.

CLMS-aware from day one

Every Grand Lodge CLMS field lives on the contact — member number, dues-paid-through date, household, raw address, USPS / CASS validation, spouse data, member-history events, officer terms, life-member dates, military service, occupation. The CLMS tab keeps it all together and the importer matches on member number to prevent duplicates.

Key features

  • Role-based access groups: Member < Officer < Secretary < Manager. Other Elks modules (elksfrs, elkssecretary, audit_log…) reference these groups directly — e.g. only Secretary can mark a dues payment as processed in CLMS.
  • De-dup on import by Member #: the importer updates the existing record (even if archived) instead of creating a duplicate.
  • Always-a-Person guard: imported members are forced to company_type = person; the name is composed from First / Middle / Last when blank.
  • Officer term history: every term a member has held, with start/end dates, partial-year flags, and a website-publishable roster.
  • Committee assignments: track who's on which committee per lodge year, with date appointed/ended and role within the committee.
  • Membership applications: full workflow from proposed → investigated → balloted → initiated, with a ballot wizard, an initiation wizard, and tracked committee assignments.
  • Volunteer ↔ HR sync: link a Volunteer contact to an hr.employee for timesheets and skills, with a duplicate-merge wizard.
  • Suspension & Drop: dedicated wizards with reason codes, date ranges, and a ribbon overlay on the contact. Reinstatement clears the flags and logs to chatter.
  • Return-to-Sender: one click flags a contact when mail comes back undeliverable. Warning banner appears on the contact until the address is updated.
  • Search filters: extra filters in Contacts — Elks Officers, Dues Paid / Overdue, Volunteers, Suspended, Dropped, Return-to-Sender.
  • Charitable activity: log volunteer hours and donations per category for Grand Lodge reporting.
  • Website officer roster: publish current officers to your lodge website automatically.

How it works

  1. Import your CLMS export or any CSV / XLSX into Contacts. Rows with a matching x_detail_member_num update the existing contact instead of creating a duplicate.
  2. The module composes the display name from First / Middle / Last when blank, and maps the title from salutation.
  3. Addresses and phones are normalized and copied into Odoo's native fields (name, title, street, city, zip, state_id, country_id, phone, mobile, fax, email); state and country are resolved by code or name.
  4. Members are kept as individuals on both create and write.
  5. Stored computed fields (x_is_member, x_is_elks_officer, x_is_dues_paid) drive the toggle visibility, search filters, and any module that depends on Elks role data.

Plays well with other Elks modules

elksfrs (Financial Reporting System) hooks into the dues fields and adds the Pay Dues workflow. elkssecretary uses the CLMS Status to drive the Secretary's queue. elksattendance, elksmaintenance, elkscharity, elksrvparking, and elkspurchase all reference the role groups defined here. Install elkscontacts alone for a directory; layer the others on as needed.

Setup

  1. Install this module from Apps.
  2. On the User form → Access Rights tab, assign the right Elks Lodge level to each user (Member / Officer / Secretary / Manager).
  3. Import your member file via Contacts → Favorites → Import. Map your column headers to the provided x_detail_* fields.
  4. After import, run Contacts → Actions → Apply Elks Mapping to refresh the native Odoo fields (name, address, phone) from the imported CLMS data.
