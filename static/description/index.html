Elks Contacts

Smarter member records for Odoo 18

Elks Contacts extends Contacts to store and maintain Elks Lodge member data on res.partner. It prevents duplicate records on import, keeps members as individuals (not companies), and maps your Elks data into native Odoo fields for cleaner day-to-day use.

Key features

  • De-dup on import by Member #: If x_detail_member_num matches an existing contact (even archived), the importer updates that record instead of creating a duplicate.
  • Always a Person: Imported/updated members are forced to company_type = person and is_company = False.
  • Field mapping helpers: Buttons/server action copy Elks fields (name parts, addresses, phones, email) into native Odoo fields (name, title, street, city, zip, state_id, country_id, phone, mobile, fax, email).
  • Officer roles: Officer position with computed type (elected/appointed) and “is officer” flag, plus a constraint to allow only one holder per role.
  • Search filters: Extra filters in Contacts (Elks Officers, Dues Paid/Overdue, Volunteers).
  • Safety net: SQL constraint ensures the Elks Member Number is unique across contacts.

How it works

  1. Import your CSV/XLSX into Contacts. When a row contains x_detail_member_num already in the database, the module merges/updates that contact instead of inserting a new one.
  2. The module composes a display name from First/Middle/Last when name is blank, and maps the title from salutation.
  3. Addresses and phones are normalized and copied into standard fields; state/country are resolved by code or name.
  4. Members are kept as individuals on both create and write.

Menus & Actions

  • Contacts → Actions → Apply Elks Mapping: runs a server action that applies the mapping helpers on the selected records. If nothing is selected, it processes all contacts that have Elks data populated.

You can run it anytime after an import to refresh native fields or to re-normalize edits.

Fields added (high level)

Numerous x_detail_* fields for member, name, address, phones, dues and household; local helper flags (x_local_*); officer role and computed flags. Native fields are updated via helper methods—no separate model is introduced.

Setup

  1. Install this module from Apps.
  2. Import your member file via Contacts → Favorites → Import. Ensure the column headers map to the provided x_detail_* fields.
  3. After import, run Contacts → Actions → Apply Elks Mapping.
